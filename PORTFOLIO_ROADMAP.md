# 포트폴리오 보강 제안 카탈로그

## Context

`my-profile-site`는 18년차 데이터 엔지니어 최우연 님의 채용/브랜딩용 단일 페이지 포트폴리오와 3편의 기술 블로그로 구성되어 있다. 최근 작업으로 H2O AutoML 채용 분석 블로그·OSS 섹션이 신설되며 콘텐츠 깊이가 한 층 올라간 상태다.

**현재 강점**
- 정량적 임팩트(9GB→3분 40초, 수십 TB, PL×2)가 한눈에 들어옴
- 기술 블로그 3편이 각각 2,800~5,200단어 규모로 의사결정 과정·트레이드오프·학술 근거까지 투명하게 서술
- Skills 섹션이 Expert/Advanced/Familiar 3단계로 숙련도를 명확히 분류
- 6개 프로젝트가 Problem–Solution–Impact 일관 구조로 제시

**관찰된 빈 자리**
- 이력서 PDF 다운로드 CTA 부재 (Contact 섹션에 이메일/LinkedIn만 노출)
- GitHub 프로필 전체 링크 미노출 (개별 레포 2개만 OSS 섹션에 링크)
- 메인 페이지 SEO·OG·sitemap·robots.txt 부재 또는 부분 적용
- 추천사·발표/세미나 등 사회적 증거(social proof) 부재
- 블로그 인덱스/태그 페이지 미구현, 정적 사이트지만 배포 URL/도메인 미확보

이번 문서는 **"무엇을 더하면 좋을지"** 의 제안 카탈로그다. 우선순위 티어별로 임팩트·소요·리스크를 명시해, 사용자가 자유롭게 골라 다음 작업으로 가져갈 수 있게 한다. 본 문서 자체는 실행 계획이 아니라 *선택지 모음*이다.

## Tier 0 — 당일~3일 / 임팩트 매우 큼 (적극 권장)

### T0-1. 이력서 PDF 다운로드 CTA 추가
- **위치 후보**: Hero 섹션의 "프로젝트 보기" 옆, Contact 섹션 카드 한 자리
- **구현**: `<a href="Data Analytics Engineer_최우연.pdf" download>` 한 줄 + 다운로드 아이콘
- **임팩트**: 채용담당자 30초 이내 의사결정에 직접 영향. INFO.md에서도 "높음" 우선순위
- **소요**: 30분
- **리스크**: 없음 (이미 파일 보유)

### T0-2. GitHub 프로필 전체 링크 노출
- **위치 후보**: Hero의 Skills Preview 카드 또는 Contact 섹션, OSS 섹션 헤더 우측
- **구현**: `https://github.com/wooyeon83` GitHub 옥토캣 SVG 버튼
- **임팩트**: 개별 레포 외 사용자의 전체 활동 그래프 노출 → 신뢰도↑
- **소요**: 15분

### T0-3. SEO·OG 메타 태그 강화 (index.html)
- **현 상태**: index.html 헤더에 일부 OG 태그가 있으나, `og:image`, `og:url`, Twitter card image, canonical URL 등이 비어 있을 가능성 큼
- **구현**:
  - `og:image` 1080×600 PNG (그라디언트 + 이름 + 직군 한 줄로 자체 제작 가능)
  - `og:url`, `twitter:image`, `canonical`
  - `theme-color` 메타
- **임팩트**: 카카오톡/링크드인/슬랙 공유 시 썸네일 노출, 검색 엔진 인덱싱 개선
- **소요**: 1~2시간 (OG 이미지 디자인 포함)

### T0-4. sitemap.xml + robots.txt 생성
- **구현**: 정적 사이트라 수동 작성 가능 (페이지 4개: index + 블로그 3개)
- **임팩트**: 검색 노출 기반 마련. T0-5 배포와 함께 가야 효과 발생
- **소요**: 30분

### T0-5. 배포 + 공개 URL 확보
- **선택지**:
  - GitHub Pages (무료, `wooyeon83.github.io/my-profile-site`)
  - Vercel (무료, 커스텀 도메인 무료, 자동 배포)
  - Cloudflare Pages
- **권장**: Vercel — Next.js 마이그레이션 시에도 그대로 활용 가능
- **임팩트**: 이력서·LinkedIn에 클릭 가능한 단일 URL 노출 가능
- **소요**: 1시간 (계정 연동 + 배포 확인)
- **선결 조건**: 없음 (현 정적 HTML 그대로 배포 가능)

## Tier 1 — 1~2주 / 콘텐츠·신뢰도 보강

### T1-1. 프로젝트 카드에 "기술 선택 근거" 줄 추가
- **현 상태**: 6개 카드가 Problem–Solution–Impact 3줄로만 구성
- **개선**: 4번째 줄 "Why this stack" 추가 (예: "MWAA — 보안 검토 통과한 매니지드 환경 + Airflow 2.10 호환")
- **임팩트**: "이 사람은 도구를 왜 골랐는지를 안다"는 인상 강화
- **소요**: 카드당 10분 × 6 = 1시간

### T1-2. 미니 아키텍처 다이어그램 (프로젝트 카드 또는 모달)
- **선택지 A (가벼움)**: 카드 하단에 1줄 텍스트 다이어그램 (`SFTP → S3 → Glue → Redshift → Tableau`)
- **선택지 B (중간)**: 카드 클릭 시 모달로 SVG 다이어그램 노출
- **선택지 C (무거움)**: 프로젝트별 상세 페이지 신설
- **권장**: A로 시작, 가치 검증되면 B
- **소요**: A=2시간 / B=1~2일

### T1-3. Testimonials(추천사) 섹션 신설
- **위치**: Experience와 Projects 사이 또는 Education 앞
- **콘텐츠 수집**: LinkedIn 추천서 1~2개 + 직접 협의로 받은 한 단락
- **포맷**: 카드형, 3개까지가 적정 (그 이상은 늘어져 보임)
- **임팩트**: 사회적 증거 — 시니어급 포트폴리오에서 강한 차별점
- **소요**: 디자인 2시간, 콘텐츠 수집은 1~2주 비동기
- **리스크**: 추천인 실명·회사명 노출 동의 필요

### T1-4. Career Highlights 수치 시각화 강화
- **현 상태**: 4개 카드 텍스트 위주
- **개선**: SVG/Chart.js로 미니 그래프 1~2개 (예: 년도별 누적 처리량 / 도메인별 프로젝트 분포 도넛)
- **임팩트**: 시각적 임팩트 + "데이터 사람"이라는 메타 메시지
- **소요**: 1일

### T1-5. 블로그 인덱스/태그 페이지
- **현 상태**: Writings 섹션에 3개 카드 수동 나열
- **개선**:
  - 별도 `writings.html` 또는 인덱스 확장
  - 태그(`Performance`, `Governance`, `AI/ML`, `PoC`) 클릭 시 필터
- **임팩트**: 블로그가 늘어났을 때 자연스럽게 확장 가능한 구조
- **소요**: 4~6시간 (정적 JS로 충분)

### T1-6. About 섹션에 "Why Data Engineering" 짧은 스토리
- **현 상태**: 약식 소개
- **개선**: 200~300자 단락 — 18년 전 시작 계기, 무엇이 흥미로웠는지, 지금 무엇에 집중하는지
- **임팩트**: 기술 능력 위에 *인격적 브랜드*를 입힘
- **소요**: 1~2시간 (글쓰기)

## Tier 3 — 사이트 진화 (선택)

### T3-1. Next.js / Astro로 마이그레이션
- 현 정적 HTML이 한 파일에 1,200줄+로 커지면서 유지보수 비용 증가 가능
- Astro가 정적 사이트 + 마크다운 블로그에 가장 가성비 좋음
- 블로그 5편 이상이 되면 권장

### T3-2. 라이트 테마 토글
- 현 다크 테마 고정. 토글은 채용담당자 취향 차이를 흡수하는 데 효과
- CSS 변수 이미 잘 정의되어 있어 비교적 쉬움 (~3시간)

### T3-3. 영문 버전 (i18n)
- 글로벌·원격 포지션을 노린다면 가치
- 한 번에 풀 i18n보다 영문 단일 페이지 `/en` 신설이 현실적

### T3-4. 방문자 분석
- Plausible (유료, 프라이버시) 또는 Cloudflare Web Analytics (무료, 가벼움)
- GA4는 데이터는 풍부하나 무거움. 포트폴리오 트래픽엔 과함

### T3-5. RSS 피드 / 검색 / "Now" 페이지
- 블로그 5편 이상부터 의미. 그 전엔 과투자

- 최근 읽은 논문·도서·기술 표 등 3~5개
- 시니어의 *지속 학습* 시그널을 보여주는 강한 신호

### T4-3. 자격증/수상 카드 시각화
- 현재 텍스트 위주. 자격증 로고(AWS, ADsP)와 함께 카드형으로 정리
- 학사/석사 프로필 사진 옆에 학교 로고 같은 작은 디테일

### T4-4. "프로젝트 임팩트" 인터랙티브 그래프
- 6개 프로젝트의 처리량·대기시간 단축률을 가로 막대 차트 한 장으로
- 각 막대 클릭 시 해당 프로젝트로 스크롤

```
[당장 1~2일]   T0-1 이력서 다운로드 → T0-2 GitHub 프로필 → T0-5 배포
[그 다음 1주] T0-3 SEO/OG → T0-4 sitemap → T1-1 기술선택 근거 → T1-6 About 스토리

[필요 시]    T3·T4 단계로 확장
```

## 결정이 필요한 갈래

1. **배포 타깃**: GitHub Pages vs Vercel vs Cloudflare Pages — 도메인을 살 의향이 있는지에 따라 갈림
2. **Next.js/Astro 마이그레이션 시점**: 지금 vs 블로그 5편 누적 후
3. **추천사 수집 의향**: 진행 가능 여부 (외부 협조 필요)
4. **영문 버전 우선순위**: 국내 vs 글로벌 채용 어느 쪽 비중을 둘지
